# Changelog

## v0.4.0 - 2026-05-31

- AI 聊天兼容性修复：当 OpenAI 兼容接口在携带 `tools` 时返回空文本且没有工具调用，聊天链路会自动用同一轮消息关闭 tools 重试，避免误显示“模型本轮未返回可展示内容”。
- 流式请求更稳健：仅在实际存在工具时发送 `tools` / `tool_choice`，降低不同模型供应商对 tool calling 支持不完整时的兼容风险。
- 补充回归测试：覆盖聊天模式 tools 空响应后的无工具降级路径，并验证既有聊天 API、会话流和后端测试集不回归。

## v0.3.9 - 2026-05-23

- 自动化运行链路重构：拆分 `run_service`、`run_query_service`、`automation_session_service`、`settings_service` 等后端服务，补齐事件、领域与路由结构，降低 `aniu_service` 耦合度，便于后续维护与扩展。
- DeepSeek thinking mode 兼容修复：在工具调用场景中保留并回传 `reasoning_content`，修复官方接口多轮对话时出现的 400 参数错误。
- 持久会话支持一键删除：聊天侧栏新增持久会话删除入口，可清空自动化上下文历史与压缩摘要，同时保留单例会话结构，避免运行记录引用失效。
- 账户总览与刷新体验优化：账户页补充交易成功率、总收益率、当日收益率与今日交易次数，新增手动刷新冷却控制，减少高频请求对上游接口的压力。

## v0.3.8 - 2026-04-23

- 自动化上下文压缩修复：压缩判定现在只统计尚未归档的有效历史消息，避免刚完成一次上下文压缩后，仅执行一轮新的盘前分析就再次触发压缩。
- 自动化 token 估算修复：压缩摘要不再被重复计入上下文估算，减少定时任务中因估算偏大导致的过早压缩。
- 补充自动化会话回归测试：新增针对压缩后历史范围与摘要计数的测试，覆盖重复压缩的回归场景。

## v0.3.7 - 2026-04-22

- 技能运行时与技能管理重构：新增统一的 skill catalog / policy / runtime / context 提供层，运行时技能与标准技能在管理页中分层展示，启停与删除权限更清晰。
- 自动化上下文策略升级：功能设置新增“最大上下文”配置，默认提升到 128K，后端自动按用户设置值的 85% 作为上下文压缩触发预算。
- 持久会话体验优化：移除了顶部固定摘要展示，只有在自动化上下文实际发生压缩时，才会在持久会话中插入“上下文压缩摘要”系统消息，且该消息不会再回注模型上下文。
- 提示词与压缩逻辑减噪：删除无 handler 技能提示、历史工具摘要和恢复摘要，统一使用“上下文压缩摘要”标签，减少定时任务上下文污染。

## v0.3.6 - 2026-04-21

- 聊天附件能力增强：支持将图片以内联数据发送给模型，并提取 `docx`、`xlsx`、`pptx` 与常见文本附件正文参与对话；同时允许仅发送附件消息，自动补齐附件型会话标题。
- 附件限制更清晰：移除了 PDF 与旧版 Office `doc/xls/ppt` 上传支持，补充运行时 `read_file` 的纯文本边界，避免把二进制文件误当文本读取。
- 分析任务详情优化：原始工具预览新增完整内容接口，前端在详情面板可按需加载被截断的原文，减少长结果查看时的信息丢失。
- 持久会话与界面细节调整：持久会话摘要标签更明确，聊天侧栏空态展示更稳定，账户总览累计表现改为总收益金额，左上角增加前端版本号显示。
